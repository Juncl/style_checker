局部拓扑强锚点匹配框架图

这张图展示的是当前节点匹配里“强锚点 + 周边节点”的工作方式：先拿到高置信锚点，再基于区域上下文和相对位置关系去推断周边节点，最后通过一对一全局收口输出稳定配对结果。

输入
设计侧节点 / 开发侧节点

包含文本、样式、坐标、层级路径、绘制顺序、可见性与视觉特征。

目标：把两侧节点整理成统一结构，作为后续匹配的基础输入。
预处理
过滤与归一化

过滤空文本、透明节点、Blank、弱可见节点；同时完成文本归一化和坐标标准化。

对应规则：可见性优先，先排除不该参与比较的节点。
锚点生成
Pass 1 高置信文本匹配

通过同文本、样式相似、位置接近等条件，先找出少量强锚点。

输出：strongAnchors / topologyAnchors。
区域上下文
区域切分与配对

先把页面切成局部区域，再通过区域中心、文本集合、装饰密度和视觉特征建立区域映射。

作用：缩小搜索空间，让拓扑只在局部块内推断。
周边锚点
nearbyAnchors

为待匹配节点选取附近锚点；短中文标签可保留多个邻近锚点，普通节点只保留最近稳定锚点。

作用：给每个节点提供局部参照系。
拓扑打分
相对关系 + 语义 + 区域亲和力

比较“相对锚点的 dx / dy / w / h”，再结合文本语义、角色、尺寸比例、IoU 与区域得分。

核心：topologyMatchScore。
候选收窄
candidatePool

优先使用同区域候选，局部区域内先完成配对，再回退到全局候选。

作用：减少跨区域串位与远距离误配。
全局收口
一对一冲突消解

把所有局部推断结果按置信度、锚点权重、拓扑分和类型优先级排序，最终只保留一对一配对。

输出：最终 pairs、unmatchedDesign、unmatchedArkui。

强锚点的来源

  • 同文本精确匹配
  • 样式相似且位置接近
  • 不是弱可见文本，也不是高风险短数字

周边节点的推断依据

  • 相对锚点位置关系
  • 局部区域上下文
  • 文本语义、角色、样式与尺寸

最终收口原则

  • 全局一对一，不允许多对一
  • 高置信优先，低置信让位
  • 局部正确，不代表全局可直接接受
