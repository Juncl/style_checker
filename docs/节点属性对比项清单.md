## 2. 详细说明

### 2.1 `fontSize`

- 适用节点：文本节点。
- 对比内容：设计稿字号 `fontSize` vs ArkUI 字号 `fontSize`。
- 特殊规则：
  - 如果 ArkUI 节点的原始类型是 `titlebar`，当前会跳过字号对比。
  - 只在两侧字号都存在时参与比较。
- 阈值：
  - 绝对差值 `> 0.5` 才产生差异。
  - 差值 `> 1.5` 时标记为 `error`，否则为 `warning`。
- 报告展示：
  - 设计值：`{字号}dp/vp`
  - 实现值：`{字号}dp/vp`
  - 描述通常是“字号偏差 +X.Xvp”。

### 2.2 `fontWeight`

- 适用节点：文本节点。
- 对比内容：设计稿字重 vs ArkUI 字重。
- 特殊规则：
  - 设计侧 `0`、空值、未显式配置的字重会被视为缺失，不会误当成 `Regular`。
  - ArkUI 枚举会先归一化成数值再对比。
- 阈值：完全不等即报差异。
- 严重级别：始终是 `error`。
- 报告展示：直接显示两侧字重值，描述为“字重不匹配”。

### 2.3 `fontColor`

- 适用节点：文本节点。
- 对比内容：设计稿颜色 vs ArkUI 颜色。
- 特殊规则：
  - 两侧都为完全透明色时，不报差异。
  - 若一侧缺失颜色，直接按“一侧缺失”处理。
- 阈值：
  - 颜色距离采用 `ΔE` 近似值。
  - `ΔE > 8` 才进入差异报告。
  - `ΔE > 40` 为 `error`，否则为 `warning`。
- 报告展示：
  - 会同时展示原始色值和可读色值。
  - 描述通常是“颜色不匹配 ΔE≈N”。

### 2.4 `fontFamily`

- 适用节点：文本节点。
- 对比内容：设计稿字体 vs ArkUI 字体。
- 特殊规则：
  - 使用字体别名表判断等价字体，例如 `HarmonyHeiTi` 与 `HarmonyOS Sans` 视为等价。
  - 任一侧缺失时不报差异。
- 阈值：只要不等价就报差异。
- 严重级别：`warning`。
- 报告展示：描述为“字体不匹配”。

### 2.5 `blur`

- 适用节点：文本节点、非文本节点。
- 对比内容：两侧统一后的模糊值。
- 特殊规则：
  - 当前报告里输出键名是 `blur`，不是 `backdropBlur`。
  - 设计侧和 ArkUI 侧都没有值时不报。
  - 两侧都为 `0` 时不报。
- 阈值：绝对差值 `> 2` 才报差异。
- 严重级别：`warning`。
- 报告展示：描述为“模糊偏差 X.X”。
- 备注：
  - 设计解析里区分了 `background` blur 和 `filter` blur，但当前比较阶段没有分别输出两个键。

### 2.6 `shadow`

- 适用节点：文本节点、非文本节点。
- 对比内容：投影是否缺失。
- 特殊规则：
  - 目前只在“设计侧有 shadow、ArkUI 侧没有 shadow”时输出。
  - 反向情况暂不输出。
- 阈值：只要出现一侧缺失就报差异。
- 严重级别：`warning`。
- 报告展示：设计值里会显示 `radius:...`，实现值为 `—`。
- 备注：
  - 当前 `shadow` 只承担“投影缺失”提示，尚未逐字段比较颜色或偏移量。

### 2.7 `shadow.radius`

- 适用节点：文本节点、非文本节点。
- 对比内容：投影模糊半径。
- 特殊规则：
  - 只有当两侧都存在 `shadow` 时才继续比较。
- 阈值：半径差值 `> 2` 才报差异。
- 严重级别：`warning`。
- 报告展示：描述为“投影模糊半径偏差”。

### 2.8 `opacity`

- 适用节点：文本节点、非文本节点。
- 对比内容：透明度。
- 特殊规则：
  - 只有两侧都能解析成数字时才比较。
  - 任一侧为 `0` 时直接忽略，不进入差异报告。
- 阈值：绝对差值 `> 0.02` 才报差异。
- 严重级别：`warning`。
- 报告展示：描述为“透明度偏差”。

### 2.9 `backgroundColor`

- 适用节点：非文本节点。
- 对比内容：填充。
- 特殊规则：
  - 以下节点类型会直接忽略填充对比：
    - `button`
    - `path`
    - `divider`
    - `progress`
    - `swiperindicator`
    - `icon`
    - `image`
    - `img`
    - `video`
    - `canvas`
  - 两侧都是透明色时视为相等。
  - 一侧透明、一侧缺失，也视为相等。
- 阈值：
  - 颜色距离 `ΔE > 8` 才报差异。
  - `ΔE > 40` 为 `error`，否则为 `warning`。
- 报告展示：描述为“填充不匹配 ΔE≈N”。

### 2.10 `borderRadius`

- 适用节点：非文本节点。
- 对比内容：四个角的圆角值。
- 特殊规则：
  - 圆形/椭圆类节点会跳过圆角对比。
  - 如果一侧有圆角、一侧没有，也会报差异。
  - 会先根据节点矩形裁剪出“可渲染最大圆角”，再比较有效值。
- 阈值：任一角差值 `> 1` 才报差异。
- 严重级别：`warning`。
- 报告展示：统一显示四角格式，例如 `12/12/12/12vp`。

### 2.11 `borderWidth`

- 适用节点：非文本节点。
- 对比内容：描边宽度。
- 特殊规则：
  - 两侧都没有描边宽度时不报差异。
  - 一侧缺失、另一侧存在时会报差异。
- 阈值：
  - 差值 `> 0.5` 才进入报告。
  - 差值 `> 2` 时为 `error`，否则为 `warning`。
- 报告展示：描述为“描边宽度偏差 X.X”或“描边宽度：一侧缺失”。

### 2.12 `borderColor`

- 适用节点：非文本节点。
- 对比内容：描边颜色。
- 特殊规则：
  - 与 `borderWidth` 同源，属于 `border` 拆分后的颜色部分。
  - 一侧缺失时按颜色缺失处理。
- 阈值：
  - 颜色比较逻辑与 `fontColor` 相同。
  - `ΔE > 8` 才报差异。
- 严重级别：按颜色距离分为 `warning` / `error`。
- 报告展示：通常标记为“描边颜色”。

### 2.13 `itemSpacing`

- 适用节点：非文本节点，且设计侧存在容器主轴间距。
- 对比内容：容器内部子元素间距。
- 特殊规则：
  - 当前是设计驱动型对比，设计侧没有 `itemSpacing` 时不会生成报告。
  - 如果设计侧有值、ArkUI 侧缺失，则报“实现缺失”。
- 阈值：
  - 差值 `> 1` 才报差异。
  - 差值 `> 4` 为 `error`，否则为 `warning`。
- 报告展示：描述为“元素间距偏差 X.X”或“元素间距：实现缺失”。

### 2.17 `spacing.horizontal`

- 适用节点：已匹配成功的兄弟节点。
- 对比内容：横向排列时的兄弟节点间距。
- 特殊规则：
  - 只在同一父节点下、且兄弟节点已全部匹配后生成。
  - 主要关注相邻兄弟之间的 `gap` 和交叉轴偏移。
- 阈值：
  - `gapDelta > 0.015` 或 `alignDelta > 0.02` 时产生报告。
  - `gapDelta > 0.03` 时为 `warning`，否则为 `info`。
- 报告展示：描述为“兄弟节点横向间距偏差 X.Xvp”。
- 额外信息：会携带两侧节点的关联矩形，供前端双层高亮。

### 2.18 `spacing.vertical`

- 适用节点：已匹配成功的兄弟节点。
- 对比内容：纵向排列时的兄弟节点间距。
- 特殊规则：与 `spacing.horizontal` 相同，只是方向相反。
- 阈值：同上。
- 报告展示：描述为“兄弟节点纵向间距偏差 X.Xvp”。

### 2.19 `alignment.horizontal`

- 适用节点：已匹配成功的兄弟节点。
- 对比内容：横向排列时的横向对齐偏差。
- 特殊规则：
  - 只在兄弟节点的交叉轴偏移超过阈值时输出。
- 阈值：交叉轴偏移 `> 0.03` 才报。
- 严重级别：`info`。
- 报告展示：描述为“兄弟节点横向对齐偏差 X.Xvp”。

### 2.20 `alignment.vertical`

- 适用节点：已匹配成功的兄弟节点。
- 对比内容：纵向排列时的纵向对齐偏差。
- 特殊规则：与 `alignment.horizontal` 相同，只是方向相反。
- 阈值：交叉轴偏移 `> 0.03` 才报。
- 严重级别：`info`。
- 报告展示：描述为“兄弟节点纵向对齐偏差 X.Xvp”。

### 2.21 `spacing.parent-child`

- 适用节点：已匹配成功的父子节点。
- 对比内容：父容器与子节点之间的内缩/边距关系。
- 特殊规则：
  - 只有父节点和子节点都已匹配时才会生成。
  - 会同时输出四个方向的 inset 值。
- 阈值：
  - `insetDelta <= 0.02` 时忽略。
  - `0.02 < insetDelta <= 0.05` 时为 `info`。
  - `insetDelta > 0.05` 时为 `warning`。
- 报告展示：描述为“父子层内边距/对齐偏差 X.Xvp”。

## 3. 报告里的显示方式

差异报告 UI 会把上面的 `property` 再映射到分类标签中，便于筛选：

- 字号：`fontSize`
- 字体：`fontFamily`
- 字重：`fontWeight`
- 颜色：`fontColor`
- 填充：`backgroundColor`
- 描边颜色：`borderColor`
- 圆角：`borderRadius`
- 阴影：`shadow`、`shadow.radius`
- 模糊：`blur`
- 不透明度：`opacity`
- 描边宽度：`borderWidth`
- 间距：`itemSpacing`、`spacing.horizontal`、`spacing.vertical`、`spacing.parent-child`
