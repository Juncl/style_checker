# 匹配阶段细则

节点匹配阶段将解析流水线输出的两侧 `UnifiedNode[]` 两两配对，生成 `MatchPair[]`。入口为 `server/src/matchers/nodeMatcher.js` → `matchNodes()`。

---

## 一、匹配方向

```
matchNodes(designNodes, arkuiNodes, { primarySource })
```

| 模式 | 触发条件 | 说明 |
|---|---|---|
| `arkui-first`（默认） | `primarySource` 缺省或 `'arkui'` | 先以 ArkUI 为主跑 design-first，翻转结果；再以 Design 为主补充；两组合并后 `selectOneToOnePairs` 裁决 |
| `design-first` | `primarySource === 'design'` | 直接执行 `matchNodesDesignFirst()`，以 Design 为遍历主序 |

**arkui-first 的实现细节**：

1. 调用 `matchNodesDesignFirst(arkuiNodes, designNodes)`，将参数对调，当作"design-first(arkui当design)"跑
2. 将结果的 `design / arkui` 字段互换，得到正向 primaryPairs
3. 再以正常方向跑 `matchNodesDesignFirst(designNodes, arkuiNodes)` 得 fallbackPairs
4. 去重合并：primary 已占的 id 不再进入 fallback
5. 合并后统一经 `selectOneToOnePairs` 最终裁决一对一

---

## 二、节点可见性判断（前置过滤）

所有 Pass 的遍历均先调用 `isMatchableNode()`，不满足条件的节点不进入匹配。

### isMatchableNode

```js
node.visible &&
!isBlankNode(node) &&
!isHiddenByFramework(node) &&
!node.visualOccluded &&
node.rect.w > 4 &&
node.rect.h > 4 &&
!isRootSizedOrLargerNode(node)   // normRect.w >= 0.999 && normRect.h >= 0.999
```

- `isHiddenByFramework`：仅当 `hiddenFrameworkAncestor=true` 时生效。文本节点直接排除；容器节点在 `pixelInvisible=true` 或 `visiblePixelRatio < 0.08` 时排除
- `isRootSizedOrLargerNode`：归一化宽高均 ≥ 0.999 的全屏包裹层不参与匹配

### isComparableOutputNode（最终输出过滤）

```js
isMatchableNode(node) && !isStructuralContainer(node)
```

- `isStructuralContainer`：`type='container'` 且无视觉装饰（`hasVisualDecoration` 返回 false）且 rawType 不在内在视觉集合中
- 这意味着纯布局容器（无背景色/圆角/边框/阴影的 FRAME/Row/Column 等）**不出现在 unmatchedDesign / unmatchedArkui** 统计中，也不参与覆盖率评分

### isAcceptablePair（配对合法性）

在所有 Pass 的输出进入 `selectOneToOnePairs` 前过滤。关键规则：

| 条件 | 处置 |
|---|---|
| 设计侧文本 + ArkUI 非文本 | 拒绝 |
| number 类型跨 matchType 时，双侧 textFieldType 必须相同 | 拒绝 |
| 短模糊数字（`isAmbiguousShortNumberText`）且不同行（`isNearSameLineSlot` 失败） | 拒绝 |
| 设计侧是 `isStructuralContainer` 且 matchType 不是 `list-index` | 拒绝 |
| 弱匹配（anchor-topology / rescue-iou / container-iou / container-geometry）且最小尺寸比 < 0.22 | 拒绝 |
| 低置信度 + 中心距 > 0.08 + 最小尺寸比 < 0.70 的 container 弱匹配 | 拒绝 |

## 三、区域切割（regionContext）

在 Pass 1 完成后，由强锚点结果构建 `regionContext`。

### segmentRegions

按 Y 轴中心升序排列，相邻节点中心间距 > 0.075（归一化）且当前区域已有 ≥4 个节点时切断，产生新区域。每个区域记录：包围盒 `rect`、节点 id 集合、`text/container` 计数、可见文本集合、装饰节点比例。

### regionMatchScore

两侧区域两两打分（score > 0.28 才入池），按分数降序一对一贪婪分配：

```
anchorVotes * 0.30    // 已匹配强锚点落在该区域对的数量（最多算 1）
geometryScore * 0.22  // 区域中心 Y 距离 + 高度比
histScore * 0.18      // text/container 计数余弦相似度
textScore * 0.10      // 区域内文本集合 Jaccard 相似度
decorationScore * 0.20 // 装饰节点比例差
```

### candidatePool

为某个设计侧节点查找 ArkUI 候选：
1. 先取该节点所属设计区域对应的 ArkUI 区域内的节点（过滤 predicate）
2. 若区域内无候选，回退到全量节点（全局兜底）
3. 返回结果为"区域内候选 + 区域外补充"的有序列表，区域内候选排在前面

## 四、Pass 详细说明

### Pass 1：文本内容精确匹配（text-content）

**触发**：设计侧 `type='text'`，`normalizeText(textContent)` 非空，且不是像素弱重复节点

**候选**：ArkUI 文本索引（`buildTextIndex`，按 `normalizeText(content)` 分桶），取文本完全一致的候选

**最优候选选取（bestExactTextCandidate）**：
```
yScore   = 1 - yDistance / 0.40   权重 0.34
xScore   = 1 - xDistance / 0.35   权重 0.26
style    = textStyleSimilarity     权重 0.22
pixel    = min(1, visiblePixelRatio / 0.14)  权重 0.18（无 samples 时默认 0.70）
```

**Y 轴接受阈值（yThreshold）**：

| 文本类型 | yThreshold | xThreshold |
|---|---|---|
| 单位文本（`isAmbiguousUnitText`） | 0.035 | ∞ |
| 短模糊数字（`isAmbiguousShortNumberText`） | 0.045 | 0.10 |
| 全局唯一 + styleScore ≥ 0.70 | 0.34 | ∞ |
| xDist < 0.08 + styleScore ≥ 0.70 | 0.22 | ∞ |
| 其他 | 0.14 | ∞ |

**强锚点（strongAnchor）判定**：不是短模糊数字 + yDistance < 0.05 + styleScore ≥ 0.78 → `confidence='high'`，进入 `strongAnchors` 和 `topologyAnchors`

**像素弱重复过滤（isWeakPixelDuplicate）**：若某文本节点的 `visiblePixelRatio < 0.08` 且同侧存在相同文本且比率 ≥ 0.10 的节点，跳过该节点

---

### Pass 2a：动态数字槽位匹配（dynamic-number-slot）

**触发**：设计侧文本 `textFieldType = 'number'`，尚未匹配

**算法（numericSlotScore）**：
```
条件：dx ≤ 0.20，dy ≤ 0.16，style ≥ 0.60，hRatio ≥ 0.50
短模糊数字额外要求：isNearSameLineSlot(dn, an, 0.10, 0.045)

xScore * 0.34 + yScore * 0.24 + style * 0.22 + hRatio * 0.10 + regionScore * 0.06 + lineScore * 0.04
```

**阈值**：score ≥ 0.70 → confidence = score > 0.82 ? 'medium' : 'low'

---

### Pass 2b：动态时间/星期槽位匹配（dynamic-text-slot）

**触发**：`textFieldType ∈ { 'time', 'weekday' }`，两侧均有此类文本

**算法（orderedSlotAssignment）**：将两侧同类型文本按阅读顺序排列后，用动态规划对齐（允许跳过）。

单对评分（slotScore）：
```
条件：dx ≤ 0.24，dy ≤ 0.22，style ≥ 0.55，hRatio ≥ 0.45

xScore * 0.24 + yScore * 0.18 + orderScore * 0.24 + style * 0.16 +
hRatio * 0.10 + regionScore * 0.04 + lineScore * 0.04 + 0.72 * 0.04
```

**阈值**：单对 score ≥ 0.58 → confidence = score > 0.72 ? 'medium' : 'low'

---

### Pass 2c：同行文本行槽位匹配（text-row-slot）

**触发**：两侧各自存在同行（centerY 差 < 0.018）且含 ≥2 个节点的文本行，行内文本满足 `isRowSlotText`（`time/weekday` 类型或 2~4 字纯汉字），且 centerY > 0.26（排除顶部状态栏）

**行匹配（bestRow）**：
```
条件：dy ≤ 0.08，得分 ≥ 0.52

(1 - dy/0.08) * 0.45 + countScore * 0.20 + xOverlapScore * 0.35
```

确认行对后，用 `orderedSlotAssignment` 对齐行内节点（score ≥ 0.50 才记录）

**ArkUI 侧额外过滤**：`isSlotVisibleEnough`：`visiblePixelRatio ≥ 0.12` 或 `textStrokeScore ≥ 0.09`

---

### Pass 3：区域内文本最优匹配（region-text-optimal）

**触发**：当 regionContext 存在时，按区域对逐对处理未匹配文本节点

**核心算法（maxWeightTextMatching）**：

构建二部图，每条边评分：
```
semanticOrRole * 0.38 + style * 0.24 + geom * 0.22 + order * 0.10 + regionScore * 0.06
```
- `semanticOrRole = max(semantic, roleScore * 0.72, fingerprintGeomOk ? 0.66 : 0)`
- `geom = xScore * 0.40 + yScore * 0.60`（xLimit=0.28，yLimit=0.08）
- `order`：两侧文本的 Y/X 排序差距（[0,1]）
- 最大权值匹配：双侧 ≤18 节点时用 bitmask DP，否则贪婪

**前置过滤**：
- 弱可见文本（`visiblePixelRatio < 0.10 && textStrokeScore < 0.08`）须通过 `weakVisibleTextCompatible`（centerDist ≤ 0.16，xDist ≤ 0.12，yDist ≤ 0.12，且语义/角色达标）
- `passesTextSemanticGate` 或 `roleScore ≥ 0.85` 或 `fingerprintGeomOk` 至少一项满足
- 单位文本：Y 距离 ≤ 0.04
- 短模糊数字：`isNearSameLineSlot(dn, an, 0.10, 0.045)`

**阈值**：score ≥ 0.58；confidence = score > 0.74 ? 'medium' : 'low'

**兜底（region-text-global-rescue）**：区域内匹配结束后，若仍有剩余未匹配文本，全局再跑一次，score ≥ 0.60

---

### Pass 3.5：文本语义角色匹配（text-role）

**触发**：设计侧文本未匹配，`hasUsableText` 满足

**算法（textRoleMatchScore）**：
```
条件：dx ≤ 0.08，dy ≤ 0.08，style ≥ 0.72，且角色兼容（inferTextRole）

xScore * 0.30 + yScore * 0.40 + style * 0.30
```

- `inferTextRole`：按字号/字重/文本内容推断 `page-title / title / subtitle / body`；含 `hasNumericSignal` 的文本返回 `body`
- body 角色不参与 role 匹配（`textRoleMatchScore` 直接返回 0）

**接受条件**：score ≥ 0.85 或通过 `isStrongTitleSlotMatch`（page-title/title 类专用严格门槛）

---

### Pass 4：强锚点周边拓扑匹配（anchor-topology）

**触发**：存在 `topologyAnchors`，设计侧节点在某锚点 normRect 距离 < 0.35（短 CJK 标签放宽到 0.70，最多取 6 个锚点）

**算法（topologyMatchScore）**：
```
relScore * 0.38 + sizeScore * 0.18 + typeScore * 0.10 + textScore * 0.10 +
semanticScore * 0.14 + iou * 0.06 + regionScore * 0.04
```
- `relScore = 1 - relDist / 0.22`（relDist = 两侧相对位移向量的欧氏距离，> 0.24 直接跳过）
- `sizeScore`：容器取宽高比均值；文本固定 1.0
- `semanticScore`：文本对取 `max(semantic, roleScore * 0.72, slotFallback ? 0.52 : 0)`；容器固定 0.70

**前置语义门槛**：文本对须通过 `passesTextSemanticGate` 或 `roleScore ≥ 0.85` 或 `slotFallback`（短 CJK 同槽位）

**候选排序**：按 bestScore 降序，同分按与最近锚点的距离升序，再按 path 字典序；高分先消费 ArkUI id

**阈值**：score > 0.58 → confidence = score > 0.72 ? 'medium' : 'low'

---

### Pass 4.5：同行同类列表顺序匹配（list-index）

**触发条件（全部满足）**：

1. 两侧各自能识别出"同行 + 同 rawType + 相近 w（容差 0.005）+ x 不重叠"的列表，且 ≥2 个元素
2. 两侧列表整体 Y 中心差 ≤ 0.02
3. 两侧列表**上方**均存在已匹配 pair（设计侧 pair.design 的底边 ≤ 列表 top，对应 arkui 同理）
4. 两侧列表**下方**均存在已匹配 pair（设计侧 pair.design 的顶边 ≥ 列表 bottom，对应 arkui 同理）
5. 首节点 IoU ≥ 0.40

**匹配策略**：两侧列表按 x 升序对齐前 `min(N_d, N_a)` 个，`confidence='high'`，不评 IoU 直接锁定

**注意**：本 Pass **不跳过已匹配节点**，高优先级可覆盖前序 Pass 的错配

---

### Pass 5a：文本位置回退匹配（text-position）

**触发**：设计侧文本未匹配，`hasUsableText` 满足

**前置拦截（allowsTextPositionFallback）**：
- 通过 `passesTextSemanticGate` → 直接允许
- 任一侧含 `hasNumericSignal`（含数字、%、°、单位词） → **拒绝**
- 任一侧是 `isAmbiguousUnitText` → **拒绝**
- 两侧 `textFieldType` 不同 → 拒绝
- 两侧均为 label 且最短文本 ≥ 5 字符 → 允许（长描述性 mock 文本回退）

**Y 轴限制（yLimit）**：
- `passesTextSemanticGate` 通过：yLimit = **0.05**
- 否则：yLimit = **0.04**
- `dx > 0.25` 直接跳过

**评分**：
```
yScore * 0.45 + xScore * 0.20 + semantic * 0.15 + style * 0.15 + regionBonus * 0.05
```

**阈值**：score > 0.35 → confidence = 'medium'

---

### Pass 5b：数字槽位匹配（numeric-slot）

**触发**：设计侧文本未匹配，且 `textFieldType = 'number'`；ArkUI 候选也须为 `'number'`

**条件**：dy ≤ 0.08，dx ≤ 0.20，`textStyleSimilarity ≥ 0.70`

**评分**：
```
(1 - dy/0.08) * 0.55 + (1 - dx/0.20) * 0.25 + styleScore * 0.20
```

**阈值**：score > 0.40 → confidence = 'medium'

---

### Pass 5c：容器 IoU 匹配（container-iou）

**触发**：设计侧 `type='container'`，未匹配

**候选**：ArkUI 中 `type='container'` 的节点（区域过滤）

**阈值**：
- 有视觉装饰（`hasVisualDecoration`）：IoU > **0.40** → confidence = 'high'
- 无视觉装饰：IoU > **0.60** → confidence = 'medium'

**注意**：本 Pass 不标记 `usedArkui`，允许后续产生重复候选，由 `selectOneToOnePairs` 最终裁决

---

### Pass 6：视觉容器几何匹配（container-geometry）

**触发**：设计侧满足 `isRenderableNonTextNode`（type=container 且有视觉装饰或 rawType 为内在视觉类型），未匹配

**内在视觉 rawType 集合**：`image, button, search, searchfield, symbolglyph, circle, ellipse, rect, vector, boolean_operation`

**算法（bestIoUMatch）**：
```
score = iou * wRatio * hRatio * (1 + regionBonus * 0.25)
```

**阈值**：IoU > **0.55** → confidence = 'medium'

**注意**：本 Pass 不标记 `usedArkui`

---

### Pass 6.5：空间担保匹配（spatial-bracket）

**触发**：设计侧 `type='container'` 未匹配，且在已确认匹配对（`selectOneToOnePairs` 预览结果）中能找到"左右夹住"的担保对

**担保条件（全部满足）**：
1. 两个担保对的设计侧节点与目标节点 **同行**（Y 中心差 < 0.08）
2. 两个担保对的 ArkUI 侧节点也与目标节点同行（< 0.08）
3. 左担保对设计侧的右边 ≤ 目标中心 X，右担保对设计侧的左边 ≥ 目标中心 X
4. 左担保对的 ArkUI 侧也从左夹住候选，右担保对从右夹住
5. 两个担保对的设计侧互相同行（Y 差 < 0.06），ArkUI 侧同理
6. 候选节点 Y 中心差 ≤ 0.05
7. 最小尺寸比 ≥ 0.5

**评分**：`wRatio * 0.5 + hRatio * 0.5`

**confidence**：取两个担保对中较低的 confidence（`lowerConfidence`）

---

### Pass 7：Rescue 兜底（rescue-iou）

**触发**：设计侧节点未匹配，类型兼容（`isCompatibleType`）

**文本对额外限制**：
- 须通过 `passesTextSemanticGate`
- 若任一侧为弱可见文本节点，须满足 `isWeakRescueTextAllowed`：
  - number 类型须 `numericTextCompatible`
  - 非 label 类型须文本完全相同
  - label 类型须 `textSemanticSimilarity ≥ 0.55`

**阈值**：IoU > **0.25** → `confidence = 'low'`

## 五、文本语义相关函数

### normalizeText

1. 全部转小写、去首尾空格
2. 含 CJK 或 `\d°%/` 的文本：去除所有空白
3. 其他：多空格压缩为单空格

### textFieldType

按正则优先级返回类型标签：

| 类型 | 示例 |
|---|---|
| `time` | `12:30`、`下午 3:00`、`现在` |
| `weekday` | `周一`、`星期三` |
| `duration` | `3小时30分钟`、`45秒` |
| `percent` | `78%`、`-3.5%` |
| `temperature` | `23°`、`-5.5°` |
| `number` | `42`、`1024`、`3.14km`、`100步` |
| `date` | `3月15日`、`2024年1月` |
| `label` | 其他（包括含数字但不符合上述格式的混合文本） |

### textSemanticSimilarity

```
if ta === tb → 1
if both label + CJK + 共同后缀 → suffixBoost（≤1 略低于 1）
typeScore = (类型相同 && 非 label && 数字时兼容) ? 0.72 : 0
charScore = charJaccard(ta, tb)
bigramScore = bigramJaccard(ta, tb)
return max(typeScore, prefixScore, suffixScore, charScore * 0.45 + bigramScore * 0.55)
```

### textStyleSimilarity

| 字段 | 容差 | 权重 |
|---|---|---|
| fontSize | ±6（线性衰减） | 0.34 |
| fontWeight | ±400（线性衰减） | 0.22 |
| fontColor | 欧氏距离 ÷ 180 | 0.28 |
| textAlign | 不同时得 0.55 | 0.16 |

**注意**：只参与有值的字段；全部缺失时返回 0.75

### passesTextSemanticGate

允许非完全相同文本进入位置/拓扑匹配的语义门槛：

- 任一侧为空或完全相同 → true
- 任一侧为 number：双侧同类且 `numericTextCompatible`
- 双侧均含 CJK 且最短 ≤ 4 字符：semantic ≥ 0.24
- 其他 label 对：semantic ≥ 0.22

### textAttributeFingerprint

将文本节点的样式量化为可比较的"指纹"：`type | fontSize(步长1) | fontWeight(步长100) | fontColor | textAlign | letterSpacing(步长0.25)`

两节点指纹完全相同，且 Y 中心差 ≤ 0.03 且 X 中心差 ≤ 0.06 时，`fingerprintGeomOk = true`，可替代语义门槛通过 Pass 3

## 六、最终一对一选取（selectOneToOnePairs）

所有 Pass 的输出合并后过 `isAcceptablePair` 过滤，再用 `selectOneToOnePairs` 保证每个节点只出现在一个 pair 中。

**排序键（降序）**：

```
pairPriority = confidenceScore + anchorScore + matchTypePriority + topologyScore * 10 + iouScore
```

| 字段 | 取值 |
|---|---|
| confidenceScore | high=300 / medium=200 / low=100 |
| anchorScore | isAnchor=25 / 否=0 |
| matchTypePriority | 见下表 |
| topologyScore * 10 | [0,10] |
| iouScore（非文本对） | iou * 8 |

**同分裁决**：`backgroundMatchPriority`（两侧 backgroundColor 是否一致）→ Design 节点 path 字典序 → ArkUI 节点 path 字典序

**matchType 优先级表**：

| matchType | 优先级 |
|---|---|
| text-content | 40 |
| region-text-optimal | 34 |
| text-role | 30 |
| anchor-topology | 24 |
| dynamic-text-slot | 22 |
| text-row-slot | 21 |
| text-position | 20 |
| numeric-slot | 19 |
| list-index | 18.5 |
| container-iou | 18 |
| spatial-bracket | 16 |
| container-geometry | 14 |
| region-text-global-rescue | 8 |
| rescue-iou | 2 |
