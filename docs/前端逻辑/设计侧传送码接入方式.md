# 设计侧传送码接入方式

## 概述

上传文件页（`UploadPage.vue`）的设计侧提供两种获取设计数据的方式：
1. **传送码模式**（主路径）：输入传送码，由后端或平台 API 返回 design.json 和设计图片
2. **文件上传模式**（调试路径）：手动上传 JSON 和图片文件，仅在 `debugger=1` 模式下可切换

本文档重点说明**传送码**的接入位置与接入方法。

## 入口位置

### UI 展示层

**文件**：`client/src/components/UploadPage.vue`

| 模式 | 行号 | 说明 |
|---|---|---|
| 非 debugger 模式（默认） | 第 156–175 行 | 设计侧直接展示传送码输入框，无切换按钮 |
| debugger 模式（`?debugger=1`） | 第 94–113 行 | 传送码模式默认开启，可点击"切换"按钮进入文件上传模式 |

两处 UI 共用同一个 `annotationUrl` 响应式变量（第 309 行）和同一个"确认"按钮的点击处理函数 `validateAnnotationUrl`。

### 核心函数

#### `validateAnnotationUrl()`（第 354–368 行）

用户点击"确认"后触发，负责：
1. 读取 `annotationUrl` 的值
2. 调用 `checkTransmissionCode(code)` 做校验和数据获取
3. 校验不通过时弹出错误提示
4. 校验通过后（目前为预留逻辑），后续将处理返回的 json 和 image

```js
function validateAnnotationUrl() {
  const code = annotationUrl.value.trim()
  if (!code) return

  const result = checkTransmissionCode(code)

  if (!result.valid) {
    ElMessage.error('传送码不合规，请检查输入')
    return
  }

  // 后续：当合规时处理返回的 designJson 和 designImageUrl
  // console.log('合规:', result.designJson, result.designImageUrl)
}
```

#### `checkTransmissionCode(code)`（第 371–376 行）⭐ **接入点**

**这是唯一需要改造的函数。** 当前为预留空函数，所有输入均返回 `{ valid: false }`：

## 接入方法

### 第一步：改造 `checkTransmissionCode`

将传送码发送到对应平台的接口，获取设计稿数据。函数签名保持不变：

```js
async function checkTransmissionCode(code) {
  try {
    // 示例：调用传送码解析接口
    const res = await fetch('/api/design/resolve', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ code }),
    })
    if (!res.ok) return { valid: false }
    const data = await res.json()
    // 期望返回：{ valid: true, json: '...base64...', image: '...base64...' }
    return data
  } catch {
    return { valid: false }
  }
}
```

> 如果接口是同步的（例如本地解码），也可以不用 async，直接返回结果对象。

### 第二步：将返回数据写入 `uploadFiles`

`checkTransmissionCode` 返回 `{ valid: true }` 后，需要将 json 和 image 数据转换为 `File` 对象，通过 `emit('step-picked', ...)` 写入父组件的 `uploadFiles`。

在 `validateAnnotationUrl` 的注释处补充以下逻辑：

```js
if (!result.valid) {
  ElMessage.error('传送码不合规，请检查输入')
  return
}

// 将返回数据转换为 File 对象
if (result.designJson) {
  // result.designJson 是 JSON 对象，直接序列化为字符串，不经过 base64
  const jsonBlob = new Blob([JSON.stringify(result.designJson)], { type: 'application/json' })
  const jsonFile = new File([jsonBlob], 'design.json', { type: 'application/json' })
  emit('step-picked', { type: 'designJson', file: jsonFile })
}

if (result.designImageUrl) {
  // result.designImageUrl 是带前缀的 base64 Data URL，格式："data:image/png;base64,..." 或 jpg/jpeg 等
  const [meta, b64] = result.designImageUrl.split(',')
  const mimeType = meta.match(/:(.*?);/)?.[1] || 'image/png'
  const ext = mimeType.split('/')[1] || 'png'
  const imageBlob = new Blob([Uint8Array.from(atob(b64), c => c.charCodeAt(0))], { type: mimeType })
  const imageFile = new File([imageBlob], `design.${ext}`, { type: mimeType })
  emit('step-picked', { type: 'designImage', file: imageFile })
}

ElMessage.success('设计稿获取成功')
```

## 接口返回格式参考

`checkTransmissionCode` 的返回值约定（由接入方自行实现）：

| 字段 | 类型 | 说明 |
|---|---|---|
| `valid` | `boolean` | 传送码是否合规且数据获取成功 |
| `errorMsg` | `string`（可选） | 校验失败时的错误提示文案 |
| `designJson` | `object`（可选） | design.json 的 JSON 对象（非 base64，直接可用） |
| `designImageUrl` | `string`（可选） | 设计图片的 base64 Data URL（含 `data:image/...;base64,` 前缀，mime 类型可为 png/jpg/jpeg 等）|

## 注意事项

- `checkTransmissionCode` 目前是同步函数，改为 `async` 时需同步将 `validateAnnotationUrl` 也改为 `async`，并用 `await` 调用。
- `designJson` 和 `designImage` 写入 `uploadFiles` 后，"开始对比"按钮的 disabled 状态会自动解除（由 `App.vue:206` 的 computed 属性控制）。
- 传送码输入框（`annotationUrl`）在两种模式下共用，不需要分别处理。
- 文件上传模式（debugger 下切换）走完全不同的路径（`onStep3Picked` / `onStep4Picked`），与传送码逻辑互不影响。
