# 接口文档

> **说明**：所有接口均通过 `client/src/api/api.ts` 调用，基础路径由 `adminEnv.ts` 中的 `ADMIN_BASE_URL` 控制：外网为 `/mock`，内网为 `window.location.origin`。
>
> **通用请求头**：所有 7 个接口均需携带 `uiplusToken`（值来自 `localStorage.getItem('uiplusToken')`）。mock 环境不校验此字段，但内网后台依赖它进行鉴权，缺失会导致接口报权限错误。

## 通用响应结构

```json
{
  "content": <业务数据>,
  "errorCode": 200,
  "errorMessage": null,
  "success": true
}
```

## 三、页面（Page）与版本（Version）

### addConsistencyCheckPage — 新增页面 / 追加版本

| 项 | 值 |
|---|---|
| **方法** | POST |
| **路径** | `/main/rest.root/reviewTool/consistencyCheck/addConsistencyCheckPage` |
| **Content-Type** | `application/json` |
| **请求头** | `uiplusToken: localStorage.getItem('uiplusToken')` |
| **前端函数** | `addConsistencyCheckPage(params)` |
| **Mock** | ✅ 已实现 |

**双模式说明**

| `id` 字段 | 行为 |
|---|---|
| **不传**（新建模式） | 在指定交付件下新建页面，同时写入首个版本，返回新建页面 ID |
| **传字符串**（追加模式） | 在已有页面下追加新版本，页面本身不变，返回该页面 ID |

**入参（Body）**

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| id | string | 否 | 已有页面 ID；**传此字段时为追加模式** |
| deliverableId | string | 是 | 所属交付件 ID |
| name | string | 是 | 页面名称，格式 `page{N}`，N = 该交付件下当前页面总数 + 1（新建前先查 `getPagesByDeliverableId` 取 `length`）；列表按 createTime 降序展示，编号越大排越前 |
| deviceType | string | 是 | 设备类型，如 `hmPhone` |
| versionName | string | 是 | 版本名称（时间戳字符串） |
| devImageBase64Data | string | 是 | 开发侧图片 Base64 |
| devJson | string | 是 | 开发侧原始 JSON 文本 |
| designImageBase64Data | string | 是 | 设计侧图片 Base64 |
| designJson | string | 是 | 设计侧原始 JSON 文本 |
| problems | array | 是 | 差异列表，见下方结构 |

**problems 单项结构**

```ts
{
  id:   string  // `${arkuiNodeId}-${property}`
  key:  string  // 节点类型：'text' | 'container'
  type: string  // 属性名，如 'fontSize'
  desc: string  // 差异描述
  data: string  // JSON.stringify(完整 diff 对象)
}
```

**出参**

| 字段 | 类型 | 说明 |
|---|---|---|
| success | boolean | 是否成功 |
| errorCode | number | 200 = 成功 |
| errorMessage | string \| null | 错误信息，成功时为 null |
| content | number | 页面 ID（新建模式为新建 ID，追加模式为传入的 ID） |

> **新建模式调用链**：`addConsistencyCheckDeliverable()` → 得到 `deliverableId` → `addConsistencyCheckPage({ deliverableId, ... })` → 得到 `pageId`
>
> **追加模式调用链**（重新对比）：`addConsistencyCheckPage({ id: pageId, ... })` → `getResultsByPageId(pageId, 1, 999)` → 更新版本列表

### getResultsByPageId — 查询页面下的版本列表

| 项 | 值 |
|---|---|
| **方法** | GET |
| **路径** | `/main/rest.root/reviewTool/consistencyCheck/getResultsByPageId?pageId={pageId}&pageNum={pageNum}&pageSize={pageSize}` |
| **请求头** | `uiplusToken: localStorage.getItem('uiplusToken')` |
| **前端函数** | `getResultsByPageId(pageId, pageNum, pageSize)` |
| **Mock** | ✅ 已实现 |

**入参（Query）**

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| pageId | number \| string | 是 | 页面 ID |
| pageNum | number | 是 | 页码（从 1 开始） |
| pageSize | number | 是 | 每页条数 |

**出参**

| 字段 | 类型 | 说明 |
|---|---|---|
| success | boolean | 是否成功 |
| errorCode | number | 200 = 成功 |
| errorMessage | string \| null | 错误信息，成功时为 null |
| content | object | — |
| content.totalCount | number | 该页面下版本总数 |
| content.list | array | 版本列表 |
| content.list[].id | number | 版本 ID |
| content.list[].pageId | number | 所属页面 ID |
| content.list[].name | string | 版本名称（时间戳字符串） |
| content.list[].devBase64Data | string | 开发侧截图 Base64，可直接作为 `<img src>` |
| content.list[].designBase64Data | string | 设计侧截图 Base64，可直接作为 `<img src>` |
| content.list[].devJsonUrl | string | 开发侧 JSON 路径，格式 `//reviewTool/2026/check/{deliverableId}/{pageId}/{ts}_dev.json`，需通过 `fetchVersionJson(url)` 读取内容 |
| content.list[].designJsonUrl | string | 设计侧 JSON 路径，同上 |
| content.list[].problems | array | 差异列表，结构同 `addConsistencyCheckPage` 入参的 `problems` 字段 |
| content.list[].creatorId | number | 创建人 ID |
| content.list[].createTime | number | 创建时间（Unix ms） |

> 取最新版本 ID：`content.list[0].id`（调用时 `pageSize=1` 只取第一条）

### 完整调用链（对比完成后自动存档）

```
对比完成（result 拿到）
  → getTeamList()               → teamId
  → getSonListByTeamId(teamId)  → subTeamId
  → addConsistencyCheckDeliverable(subTeamId, name)  → deliverableId
  → addConsistencyCheckPage({ deliverableId, ... })  → pageId
  → getResultsByPageId(pageId, 1, 1)                 → versionId
  → URL query: { deliverableId, pageId, versionId }
```
