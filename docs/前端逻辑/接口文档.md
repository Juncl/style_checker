# 接口文档

> **说明**：所有接口均通过 `client/src/api/api.ts` 调用，基础路径由 `adminEnv.ts` 中的 `ADMIN_BASE_URL` 控制：外网为 `/mock`，内网为 `window.location.origin`。

## 通用响应结构

```json
{
  "content": <业务数据>,
  "errorCode": 200,
  "errorMessage": null,
  "success": true
}
```

---

## 一、团队/项目

### getTeamList — 获取团队列表

| 项 | 值 |
|---|---|
| **方法** | GET |
| **路径** | `/main/rest.root/workspace/team/getMyTeam` |
| **前端函数** | `getTeamList()` |
| **Mock** | ✅ 已实现 |

**入参**：无

**出参 `content`**：`{ id: null, teamId: number }[]`

---

### getSonListByTeamId — 根据团队 ID 获取子项目列表

| 项 | 值 |
|---|---|
| **方法** | GET |
| **路径** | `/main/rest.root/workspace/team/getMySonTeam?id={teamId}` |
| **前端函数** | `getSonListByTeamId(teamId)` |
| **Mock** | ✅ 已实现 |

**入参（Query）**

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| id | number \| string | 是 | 团队 ID（来自 `getTeamList` 的 `teamId`） |

**出参 `content`**：`{ id: null, teamId: number, parentTeam: number }[]`

> 返回的 `teamId` 即 `addConsistencyCheckDeliverable` 的入参 `teamId`。

## 二、交付件（Deliverable）

### getConsistencyCheckDeliverables — 查询用户交付件列表

| 项 | 值 |
|---|---|
| **方法** | GET |
| **路径** | `/main/rest.root/reviewTool/consistencyCheck/getConsistencyCheckDeliverables` |
| **前端函数** | `getConsistencyCheckDeliverables()` |
| **Mock** | ✅ 已实现 |

**入参**：无

**出参 `content`**：`ConsistencyCheckDeliverableVo[]`

```ts
{
  id:           number   // 交付件 ID
  name:         string   // 交付件名称，格式如 "2026年6月1号 17:11:58"
  createTime:   number   // 创建时间（Unix ms）
  creatorId:    number   // 创建人 ID
  filePath:     string   // 子项目 teamId，如 "302588"
  filePathName: string   // 可读路径名称
  platformType: number   // 平台类型（0=hmPhone，1=hmWatch，2=web）
}
```

---

### addConsistencyCheckDeliverable — 新增交付件

| 项 | 值 |
|---|---|
| **方法** | POST |
| **路径** | `/main/rest.root/reviewTool/consistencyCheck/addConsistencyCheckDeliverable` |
| **Content-Type** | `application/x-www-form-urlencoded` |
| **前端函数** | `addConsistencyCheckDeliverable(teamId, name)` |
| **Mock** | ✅ 已实现 |

**入参（Body）**

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| teamId | string | 是 | 子项目 teamId（来自 `getSonListByTeamId` 的 `teamId`） |
| name | string | 是 | 交付件名称，前端自动生成，格式同 `getConsistencyCheckDeliverables` 返回的 `name` |

**出参 `content`**：`number`（新建交付件 ID）

> **调用链**：`getTeamList()` → 取 `teamId` → `getSonListByTeamId(teamId)` → 取子项目 `teamId` → `addConsistencyCheckDeliverable(subTeamId, name)`

## 三、页面（Page）与版本（Version）

### addConsistencyCheckPage — 新增页面（含首个版本）

| 项 | 值 |
|---|---|
| **方法** | POST |
| **路径** | `/main/rest.root/reviewTool/consistencyCheck/addConsistencyCheckPage` |
| **Content-Type** | `application/json` |
| **前端函数** | `addConsistencyCheckPage(params)` |
| **Mock** | ✅ 已实现 |

**入参（Body）**

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| deliverableId | number | 是 | 所属交付件 ID |
| name | string | 是 | 页面名称（同交付件名称格式） |
| deviceType | string | 是 | 设备类型，如 `hmPhone` |
| versionName | string | 是 | 版本名称（时间戳字符串） |
| devImageBase64Data | string | 是 | 开发侧图片 Base64 |
| devJson | string | 是 | 开发侧原始 JSON 文本 |
| designImageBase64Data | string | 是 | 设计侧图片 Base64 |
| designJson | string | 是 | 设计侧原始 JSON 文本 |
| problems | array | 是 | 差异列表，见下方结构 |

**problems 单项结构**

```ts
{
  id:   string  // `${arkuiNodeId}-${property}`
  key:  string  // 节点类型：'text' | 'container'
  type: string  // 属性名，如 'fontSize'
  desc: string  // 差异描述
  data: string  // JSON.stringify(完整 diff 对象)
}
```

**出参 `content`**：`number`（新建页面 ID）

> **调用链**：`addConsistencyCheckDeliverable()` → 得到 `deliverableId` → `addConsistencyCheckPage(params)` → 得到 `pageId`

---

**入参（Query）**

|---|---|---|---|

---

### getResultsByPageId — 查询页面下的版本列表

| 项 | 值 |
|---|---|
| **方法** | GET |
| **路径** | `/main/rest.root/reviewTool/consistencyCheck/getResultsByPageId?pageId={pageId}&pageNum={pageNum}&pageSize={pageSize}` |
| **前端函数** | `getResultsByPageId(pageId, pageNum, pageSize)` |
| **Mock** | ✅ 已实现 |

**入参（Query）**

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| pageId | number \| string | 是 | 页面 ID |
| pageNum | number | 是 | 页码（从 1 开始） |
| pageSize | number | 是 | 每页条数 |

**出参 `content`**：`{ list: Version[], totalCount: number }`

> 取最新版本 ID：`content.list[0].id`（调用时 `pageSize=1` 只取第一条）

---

### 完整调用链（对比完成后自动存档）

```
对比完成（result 拿到）
  → getTeamList()               → teamId
  → getSonListByTeamId(teamId)  → subTeamId
  → addConsistencyCheckDeliverable(subTeamId, name)  → deliverableId
  → addConsistencyCheckPage({ deliverableId, ... })  → pageId
  → getResultsByPageId(pageId, 1, 1)                 → versionId
  → URL query: { deliverableId, pageId, versionId }
```
