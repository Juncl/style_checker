# 前端架构文档

> 本文档描述 `client/src/` 目录下前端代码的分层架构、模块职责、组件关系与数据流。

---

## 一、技术栈

| 类别 | 技术 |
|---|---|
| 框架 | Vue 3（Composition API，`<script setup>`） |
| 路由 | Vue Router 5 |
| UI 组件库 | Element Plus |
| HTTP | Axios（封装于 `api/request.ts`） |
| 构建工具 | Vite + vue-tsc |
| 语言 | JavaScript（Vue 文件）/ TypeScript（api、init、router） |

## 五、初始化流程

`ConsistencyView.vue` 的 `onMounted` 调用 `initApp()`，内部**严格串行**执行 5 步：

```
onMounted
    │
    ▼
1. checkAuth()         检查 localStorage.uiplusToken，无则调登录接口（预留）
    │
    ▼
2. mountPixso()        挂载 Pixso 渲染器（预留空函数）
    │
    ▼
3. detectIframe()      window.self !== window.top 判断，结果写入 window.__inIframe
    │
    ▼
4. restorePlatform()   读 localStorage.cachePlatform，返回上次平台 key
    │
    ▼
5. trackEntry()        URL 同时含 deliverableId / pageId / versionId 时调打点接口
    │
    ▼
返回 { platform }
    │
    ▼
currentPlatform.value = platform
fetchCases(platform)
```

## 六、数据流

### 6.1 上传文件分析

```
用户上传文件
    │ onStepPicked(type, file)
    ▼
uploadFiles.value 写入对应 slot
    │ 点击"开始对比"
    ▼
runUpload(platform)
    │ POST /api/check/upload (multipart)
    ▼
result.value = 返回数据
    │
    ▼
UploadPage 隐藏 → ReportPage 渲染
```

### 6.2 选择内置案例

```
用户点击案例
    │ emit('select-case', id)
    ▼
selectCase(id)
    │ POST /api/check/case/:id
    ▼
result.value = 返回数据 → ReportPage 渲染
```

### 6.3 平台切换

```
右侧下拉框 @change
    │ onPlatformSwitch(platform)  [UploadPage]
    ▼
emit('platform-switch', platform)
    │
    ▼
onPlatformSwitch(platform)        [ConsistencyView]
    ├── savePlatform(platform)    → localStorage.cachePlatform
    └── fetchCases(platform)      → 刷新案例列表
```

### 6.4 节点点击联动

```
ImagePanel / NodeTree 节点点击
    │ emit('arkui-node-click' / 'design-node-click', nodeId)
    ▼
ConsistencyView.onArkuiNodeClick / onDesignNodeClick
    │ 查找 pairs 中对应配对
    ▼
selectedPair.value 更新
    │ props 下发到 ImagePanel / DiffReport
    ▼
Canvas 高亮矩形 + 差异列表滚动定位
```

## 七、API 层说明

| 函数 | 方法 | 路径 | 说明 |
|---|---|---|---|
| `fetchCases(platform)` | GET | `/api/cases?platform=` | 获取内置案例列表 |
| `checkCase(id, platform)` | POST | `/api/check/case/:id` | 分析内置案例 |
| `checkUpload(...)` | POST | `/api/check/upload` | 上传文件分析 |
| `imageUrl(id, type, platform)` | — | `/api/cases/:id/image/:type` | 案例图片 URL |
| `reportTrack(params)` | POST | 待接入 | 入口埋点打点（预留） |

**请求拦截器**：`request.ts` 中 axios 实例每次请求前自动从 `localStorage.uiplusToken` 读取 token，写入 `Authorization: Bearer <token>` 请求头。
