## 五、初始化流程

`ConsistencyView.vue` 的 `onMounted` 调用 `initApp()`，内部**严格串行**执行 5 步：

```
onMounted
    │
    ▼
1. checkAuth()         检查 localStorage.uiplusToken，无则调登录接口（预留）
    │
    ▼
2. mountPixso()        挂载 Pixso 渲染器（预留空函数）
    │
    ▼
3. detectIframe()      window.self !== window.top 判断，结果写入 window.__inIframe
    │
    ▼
4. restorePlatform()   读 localStorage.cachePlatform，返回上次平台 key
    │
    ▼
5. trackEntry()        URL 同时含 deliverableId / pageId / versionId 时调打点接口
    │
    ▼
返回 { platform }
    │
    ▼
currentPlatform.value = platform
fetchCases(platform)
```

## 六、数据流

### 6.1 上传文件分析

```
用户上传开发侧 JSON
    │ onStepPicked({ type: 'arkuiJson', file })  [ConsistencyView, async]
    ▼
detectPlatformFromJson(file)
    │ 解析 JSON 根节点：deviceType / name / content.width 判断平台
    ▼
若检测到平台与当前不同：
    ├── currentPlatform.value = detected
    ├── savePlatform(detected)   → localStorage.cachePlatform
    ├── cases.value = []         → 立即清空，防止旧 case 用新平台请求图片
    └── fetchCases(detected)     → 重新加载案例列表
    │
    ▼ （等待图片上传）
用户上传开发侧图片
    │ onStepPicked({ type: 'arkuiImage', file })
    ▼
json + image 均已就绪？
    ├── 是 → triggerDevPreview(files)   调 /api 解析节点，渲染预览
    └── 否 → 等待
```

> ⚠️ **关键约束**：无论平台是什么，**必须 JSON + 图片都上传完毕**才会触发节点预览接口。旧逻辑中 web 平台只需 JSON 即触发已废弃。

```
点击"开始对比"
    │ runUpload(platform)
    ▼
POST /api/check/upload (multipart)
    ▼
result.value = 返回数据
    │
    ▼
UploadPage 隐藏 → ReportPage 渲染
```

### 6.2 选择内置案例

```
用户点击案例
    │ emit('select-case', id)
    ▼
selectCase(id)
    │ POST /api/check/case/:id
    ▼
result.value = 返回数据 → ReportPage 渲染
```

### 6.3 平台切换

```
右侧下拉框 @change
    │ onPlatformSwitch(platform)  [UploadPage]
    ▼
emit('platform-switch', platform)
    │
    ▼
onPlatformSwitch(platform)        [ConsistencyView]
    ├── currentPlatform.value = platform
    ├── savePlatform(platform)    → localStorage.cachePlatform
    ├── cases.value = []          → 立即清空，防止旧 case 图片用新平台 URL 请求（404）
    └── fetchCases(platform)      → 重新加载案例列表
```

> ⚠️ **关键顺序**：`currentPlatform` 更新后必须先清空 `cases`，再等待 `fetchCases` 异步返回。否则过渡期间旧平台的 case 会用新平台参数请求缩略图，导致 404。

### 6.4 节点点击联动

```
ImagePanel / NodeTree 节点点击
    │ emit('arkui-node-click' / 'design-node-click', nodeId)
    ▼
ConsistencyView.onArkuiNodeClick / onDesignNodeClick
    │ 查找 pairs 中对应配对
    ▼
selectedPair.value 更新
    │ props 下发到 ImagePanel / DiffReport
    ▼
Canvas 高亮矩形 + 差异列表滚动定位
```

## 九、画布缩放功能（Ctrl + 滚轮）

### 9.1 功能描述

鼠标悬停在任意一侧画布上，按住 `Ctrl` 并滚动鼠标滚轮，可对两侧画布**同步**放大/缩小。

| 操作 | 效果 |
|---|---|
| Ctrl + 向上滚动 | 以鼠标为中心放大（factor = 1.1） |
| Ctrl + 向下滚动 | 以鼠标为中心缩小（factor = 1/1.1）|
| 窗口 resize | 两侧自动重置回 1× |
| 最小/最大缩放 | 1× ~ 100× |

### 9.2 DOM 层级（`ImagePanel.vue`）

```
.img-panel
└── .img-wrapper
    └── .zoom-clip           ← 固定容器，位置稳定，用于坐标计算
        └── .zoom-layer      ← 绑定 :style="zoomStyle"，接受 CSS transform: scale()
            ├── <img>        ← 背景图片
            └── <canvas>     ← 事件监听层（click / dblclick / mousemove / wheel）
```

`.zoom-layer` 通过计算属性驱动变换：

```js
const zoomStyle = computed(() => ({
  transform: `scale(${zoomScale.value})`,
  transformOrigin: `${zoomOriginX.value}px ${zoomOriginY.value}px`,
}))
```

### 9.3 事件处理（`onCanvasWheel`）

```js
// onMounted 时注册，必须 passive: false 才能调用 preventDefault()
canvasRef.value?.addEventListener('wheel', onCanvasWheel, { passive: false })

function onCanvasWheel(e) {
  if (!e.ctrlKey) return          // 非 Ctrl 滚轮不处理
  e.preventDefault()              // 阻止浏览器页面缩放
  const factor = e.deltaY < 0 ? 1.1 : 1 / 1.1
  // 计算鼠标在 zoom-clip 内的归一化坐标（0-1）
  const rect = zoomClipRef.value.getBoundingClientRect()
  const normX = (e.clientX - rect.left) / rect.width
  const normY = (e.clientY - rect.top)  / rect.height
  applyZoom(factor, normX, normY)
  emit('zoom', { factor, normX, normY })   // 触发对侧联动
}
```

### 9.4 缩放应用（`applyZoom`）

```js
// normX/normY：鼠标相对 zoom-clip 的归一化坐标
// 将归一化坐标转为像素坐标作为 transform-origin，保证以鼠标为中心缩放
function applyZoom(factor, normX, normY) {
  zoomOriginX.value = normX * clip.clientWidth
  zoomOriginY.value = normY * clip.clientHeight
  zoomScale.value   = Math.max(1, Math.min(100, zoomScale.value * factor))
  nextTick(draw)
}
```

### 9.5 双侧联动（`ReportPage.vue`）

两个 `ImagePanel` 实例通过 `defineExpose({ applyZoom, resetZoom })` 暴露方法，`ReportPage` 交叉调用实现同步：

```
用户在开发侧画布 Ctrl+滚轮
    │ emit('zoom', { factor, normX, normY })
    ▼
onDevPanelZoom()                    [ReportPage]
    │ designPanelRef.value?.applyZoom(factor, normX, normY)
    ▼
设计侧画布同步缩放到相同比例和视角
```

```
用户在设计侧画布 Ctrl+滚轮
    │ emit('zoom', { factor, normX, normY })
    ▼
onDesignPanelZoom()                 [ReportPage]
    │ devPanelRef.value?.applyZoom(factor, normX, normY)
    ▼
开发侧画布同步缩放到相同比例和视角
```

```js
// ReportPage.vue 中的联动处理
function onDevPanelZoom({ factor, normX, normY }) {
  designPanelRef.value?.applyZoom(factor, normX, normY)
}
function onDesignPanelZoom({ factor, normX, normY }) {
  devPanelRef.value?.applyZoom(factor, normX, normY)
}
// 窗口 resize 时重置两侧缩放
function onWindowResize() {
  devPanelRef.value?.resetZoom()
  designPanelRef.value?.resetZoom()
}
```

### 9.6 关键实现细节

| 细节 | 说明 |
|---|---|
| `passive: false` | wheel 事件默认 passive，必须显式关闭才能 `preventDefault()` 阻止浏览器缩放 |
| 变换原点不平移 | 只用 `scale()`，通过 `transform-origin` 跟随鼠标，不做 `translate` |
| 使用 `zoom-clip` 而非 `canvas` | canvas 受 scale 影响，坐标不稳定；`zoom-clip` 无 transform，getBoundingClientRect 可靠 |
| Inspector 定位不受缩放影响 | `updateInspectorPos` 也基于 `zoom-clip` 的稳定坐标计算，缩放后 Inspector 仍贴近节点 |
| canvas 在 scale 层内重绘 | `draw()` 后 canvas 大小 = 图片 clientWidth/Height（已被 scale 拉伸），节点矩形按比例渲染正确 |
