# 打点机制

## 概述

打点（埋点/上报）用于跟踪用户在页面中的关键操作行为，所有打点请求统一经 fetch 发送到 `/mock/report/interaction`（外网）或内网打点接口（内网替换时）。

## 架构

- **`client/src/views/utils-inner/report.ts`** — 打点工具函数 `reportInteraction` 的实现
  - 外网：通过 `fetch('/mock/report/interaction')` POST 上报
  - 内网：该文件整体替换为内网真实打点实现
- **`mock/routes/report.js`** — mock 打点接口，仅接收参数返回 200，不做实际存储

## 已接入的打点事件

---

### 1. `selectCase` — 点击案例

点击案例列表中的案例时触发。

**name:** `selectCase`  
**event:** `selectCase`

| 字段 | 说明 | 取值示例 |
|---|---|---|
| `user` | 用户账号，从 `localStorage.getItem('userInfo')` 解析 JSON 的 `account` 字段 | `"zhangsan"` |
| `curTime` | 当前时间的 ISO 字符串 | `"2026-06-10T12:00:00.000Z"` |
| `caseName` | 案例 ID | `"case6"` |
| `platform` | 当前平台 | `"hmPhone"` / `"hmWatch"` / `"web"` |
| `isFrom` | 来源标识，iframe 内为 `hiscenario`，否则 `octo` | `"octo"` / `"hiscenario"` |

---

| `versionId` | 版本 ID | `"345678"` |
| `platform` | 当前平台 | `"hmPhone"` / `"hmWatch"` / `"web"` |
| `isFrom` | 来源标识，iframe 内为 `hiscenario`，否则 `octo` | `"octo"` / `"hiscenario"` |

---

### 3. `clickCompare` — 完成比对

## 添加新的打点事件

1. 在 `report.ts` 中已导出的 `reportInteraction` 函数只需引入使用：

```ts
import { reportInteraction } from '../utils-inner/report'

reportInteraction({
  name: '事件名称',
  event: '事件动作',
  extend: { /* 附加字段 */ },
})
```

2. 若需新增 mock 接口字段接收，无需修改 mock 路由（接口无状态，纯接收）
3. 内网接入时只需替换 `client/src/views/utils-inner/report.ts` 文件内容
