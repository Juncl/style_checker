# 内网接入指南

---

## 一、接入步骤

### 1. 切换 API 地址

打开 `client/src/api/adminEnv.ts`，注释第一行，取消第二行注释：

```ts
// export const ADMIN_BASE_URL = '/mock'
export const ADMIN_BASE_URL = window.location.origin
```

### 2. 替换所有 inner 文件

以下文件在内网均有同名实现，直接用内网版本覆盖即可，函数签名保持一致：

```
client/src/views/consistency/init/inner/
  checkAuth.ts       ← 当前：空函数        → 内网鉴权逻辑
  mountPixso.ts      ← 当前：空函数        → Pixso SDK 初始化
  detectIframe.ts    ← 当前：空函数        → iframe 环境检测
  trackEntry.ts      ← 当前：空函数        → 真实打点调用
  loadDeliverable.ts ← 当前：return null  → 交付物拉取逻辑

client/src/views/utils-inner/
  getJsonImage.ts    ← 当前：fetch /mock/design → 内网传送码解析实现
```

## 二、文件说明

### api/ 目录

两条独立调用链，禁止交叉：

```
后台管理链：adminEnv.ts → request.ts → api.ts → 业务组件
算法服务链：requestOut.ts → index.ts → 业务组件
```

| 文件 | 权限 | 说明 |
|---|---|---|
| `adminEnv.ts` | 🔴 唯一替换点 | 接入内网只改这里 |
| `request.ts` | ❌ 不动 | 后台管理 HTTP 客户端 |
| `api.ts` | ✅ 可扩展 | 新增后台管理接口在此追加，参照 `geConsistencyCheckDeliverables` 写法 |
| `requestOut.ts` | ❌ 不动 | 算法服务 HTTP 客户端 |
| `index.ts` | ✅ 可扩展 | 新增算法服务接口在此追加，参照 `checkUpload` 写法 |
| `constants.ts` | ⚠️ 谨慎 | 仅平台类型增减时修改 |

### views/consistency/init/ 目录

```
init/
  index.ts          ← 入口，不动
  restorePlatform.ts ← 平台缓存，不动
  inner/            ← 内网替换组（当前均为空逻辑）
    checkAuth.ts
    mountPixso.ts
    detectIframe.ts
    trackEntry.ts
    loadDeliverable.ts
```

| 文件 | 权限 | 说明 |
|---|---|---|
| `index.ts` | ⚠️ 谨慎 | 初始化流程入口，步骤顺序不可随意调整 |
| `restorePlatform.ts` | ✅ 可扩展 | 平台恢复逻辑，平台增减时修改 |
| `inner/*` | ✅ 接入时替换 | 整文件夹替换，函数签名保持一致 |

### views/utils-inner/ 目录

```
utils-inner/     ← 内网替换组（当前为 mock 实现）
  getJsonImage.ts
```

| 文件 | 权限 | 说明 |
|---|---|---|
| `getJsonImage.ts` | ✅ 接入时替换 | 当前调 mock，内网替换为传送码解析实现 |
