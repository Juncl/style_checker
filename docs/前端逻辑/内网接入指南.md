## 二、文件说明

### api/ 目录

两条独立调用链，禁止交叉：

```
后台管理链：adminEnv.ts → request.ts → api.ts → 业务组件
算法服务链：adminEnv.ts → requestOut.ts → index.ts → 业务组件
```

| 文件 | 权限 | 说明 |
|---|---|---|
| `adminEnv.ts` | 🔴 唯一替换点 | 接入内网只改这里 |
| `request.ts` | ✅ 接入时替换 | 后台管理 HTTP 客户端；内网版本含接口鉴权，整文件替换 |
| `api.ts` | ✅ 可扩展 | 新增后台管理接口在此追加，参照 `getConsistencyCheckDeliverables` 写法 |
| `requestOut.ts` | ❌ 不动 | 算法服务 HTTP 客户端 |
| `index.ts` | ✅ 可扩展 | 新增算法服务接口在此追加，参照 `checkUpload` 写法 |
| `constants.ts` | ⚠️ 谨慎 | 仅平台类型增减时修改 |

### views/consistency/init/ 目录

```
init/
  index.ts           ← 入口，不动
  restorePlatform.ts ← 平台缓存，不动
  loadDeliverable.ts ← URL 有参时加载历史结果，不动
  inner/             ← 内网替换组（当前均为空逻辑）
    checkAuth.ts
    mountPixso.ts
    detectIframe.ts
    trackEntry.ts
```

| 文件 | 权限 | 说明 |
|---|---|---|
| `index.ts` | ⚠️ 谨慎 | 初始化流程入口，步骤顺序不可随意调整 |
| `restorePlatform.ts` | ❌ 不动 | 平台缓存读取，平台增减时修改 constants.ts 即可 |
| `loadDeliverable.ts` | ❌ 不动 | URL 携带三参数时拉取交付件/页面/版本数据 |
| `inner/*` | ✅ 接入时替换 | 整文件夹替换，函数签名保持一致 |

### views/utils-inner/ 目录

```
utils-inner/        ← 内网接入组（替换 or 新增，当前为占位/空实现）
  getJsonImage.ts   ← 接入时替换
  report.ts         ← 接入时新增（复制过来）
  message.ts        ← 接入时新增（复制过来）
```

| 文件 | 权限 | 说明 |
|---|---|---|
| `getJsonImage.ts` | ✅ 接入时替换 | 当前调 mock，内网替换为传送码解析实现 |
| `report.ts` | ✅ 接入时新增 | 打点初始化工具，当前不存在，内网已有，复制过来 |
| `message.ts` | ✅ 接入时新增 | 跨窗口通信工具类，当前不存在，内网已有，复制过来 |
